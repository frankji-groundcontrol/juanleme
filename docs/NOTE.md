这个文件是解释这里面的文件都是干什么的

1. INIT.md
我们在这里描述了用户需求，用户的路径，这个是我们立项时候的第一个参考资料。我们在里面也提了大致的工程期望，比如技术栈，功能模块，用户角色等。这个文档是给老板和产品经理看的。
2. PLAN.md
我们在这个文件里面描述了项目的计划，包括时间线，功能模块，技术栈等。这个文档是给开发团队看的。
3. .trea/rules/project_rules
这是给TREA用的，里面描述了项目的规则，比如代码规范，git提交规范等。因为TREA之后要根据这个规矩写代码。
4. TODO.md
我们在这个文件里面描述了项目的待办事项，包括功能模块，技术实现，测试用例等。这个文档是给开发团队看的。之后我们会用TREA沿着这个文档一步步完成项目。
5. README.md
这个是介绍产品功能的，是给目标用户看的。
